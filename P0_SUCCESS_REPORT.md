### **修复 #3: Settings页面表单验证** ✅ (完全重构)

**问题**: 
- 前端缺少实时验证
- 可以输入无效数据（如端口-1、99999）
- 用户体验差

**修复内容**:

#### **3.1 集成 react-hook-form + zod**
```typescript
// 数据库表单
const dbForm = useForm<DbConnectionFormData>({
  resolver: zodResolver(dbConnectionSchema),
  mode: 'onChange',  // 实时验证
  defaultValues: { ... }
});

// LLM表单
const llmForm = useForm<LlmConfigFormData>({
  resolver: zodResolver(llmConfigSchema),
  mode: 'onChange',  // 实时验证
  defaultValues: { ... }
});
```

#### **3.2 添加验证规则** (`lib/schemas.ts`)
- ✅ 端口: 1-65535
- ✅ 主机名: 字母数字横线点
- ✅ API Key: 最少10字符
- ✅ 所有必填字段验证

#### **3.3 实时错误提示**
```typescript
{dbForm.formState.errors.port && (
  <p className="text-sm text-red-500 mt-1">
    {dbForm.formState.errors.port.message}
  </p>
)}
```

#### **3.4 智能按钮禁用**
```typescript
<Button type="submit" disabled={!dbForm.formState.isValid}>
  保存
</Button>
```

**验证**: ✅ 用户手动测试确认
- ✅ 端口 `-1` → 红色边框
- ✅ 错误提示显示："端口号必须大于0"
- ✅ **保存按钮变灰（禁用）**

**影响**:
- ✅ 数据质量大幅提升
- ✅ 用户体验显著改善
- ✅ 类型安全100%
- ✅ 减少无效请求到后端
