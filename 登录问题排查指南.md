## 🔍 问题排查步骤

### 1. 检查后端服务状态

确认后端服务正常运行:

```bash
# 访问健康检查端点
curl http://localhost:8000/health

# 应该返回:
{
  "status": "healthy",
  "database": "connected",
  "agent": "initialized",
  "memory": "initialized",
  "version": "3.0.0"
}
```

### 2. 检查数据库连接

确认数据库配置正确:

```bash
# 查看 backend/.env 文件
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=123456
DB_NAME=chatbi
```

### 3. 测试登录 API

使用 curl 测试登录接口:

```bash
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

**成功响应**:
```json
{
  "id": 1,
  "username": "admin",
  "email": "admin@example.com",
  "token": "eyJ..."
}
```

**失败响应**:
```json
{
  "detail": "用户名或密码错误"
}
```

### 4. 检查 CORS 配置

确认 CORS 配置包含前端地址:

```python
# backend/.env
ALLOWED_ORIGINS=["http://localhost:3000","http://127.0.0.1:3000"]
```

### 5. 查看后端日志

检查后端终端输出的错误信息:

```
# 正常日志
INFO:     127.0.0.1:xxxxx - "POST /auth/login HTTP/1.1" 200 OK

# 错误日志
ERROR:    Exception in ASGI application
```

## 🔧 常见错误及解决方案

### 错误 1: 500 Internal Server Error

**可能原因**:
1. 数据库连接失败
2. 用户表不存在
3. 密码哈希验证失败

**解决方案**:

#### A. 检查数据库表

```bash
# 运行创建表脚本
cd backend
..\.venv\Scripts\python.exe -c "from models import Base; from config import settings; from sqlalchemy import create_engine; engine = create_engine(settings.database_url.replace('aiomysql', 'pymysql')); Base.metadata.create_all(engine)"
```

#### B. 重新创建测试用户

```bash
cd backend
..\.venv\Scripts\python.exe create_test_user.py
```

#### C. 检查后端日志

查看详细的错误堆栈信息。

---

### 错误 2: CORS 错误

**错误信息**:
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解决方案**:

1. 检查 `backend/.env` 中的 ALLOWED_ORIGINS
2. 确保包含前端地址:
   ```
   ALLOWED_ORIGINS=["http://localhost:3000","http://127.0.0.1:3000"]
   ```
3. 重启后端服务

---

### 错误 3: 用户名或密码错误 (401)

**可能原因**:
1. 用户不存在
2. 密码错误
3. 密码哈希算法不匹配

**解决方案**:

1. 使用测试账号登录:
   - 用户名: `admin`
   - 密码: `admin123`

2. 如果仍然失败,重新创建用户:
   ```bash
   cd backend
   ..\.venv\Scripts\python.exe create_test_user.py
   ```

---

### 错误 4: 数据库连接失败

**错误信息**:
```
Can't connect to MySQL server on '127.0.0.1'
```

**解决方案**:

1. 确认 MySQL 服务正在运行
2. 检查数据库配置:
   ```bash
   # backend/.env
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=123456
   DB_NAME=chatbi
   ```
3. 测试数据库连接:
   ```bash
   mysql -h 127.0.0.1 -u root -p chatbi
   ```

## 📝 调试技巧

### 1. 查看网络请求

打开浏览器开发者工具 (F12):
1. 切换到 Network 标签
2. 执行登录操作
3. 查看 `/auth/login` 请求:
   - 请求 URL
   - 请求方法 (POST)
   - 请求头
   - 请求体
   - 响应状态码
   - 响应内容

### 2. 查看控制台错误

打开浏览器控制台 (F12 → Console):
- 查看 JavaScript 错误
- 查看 CORS 错误
- 查看网络错误

### 3. 查看后端日志

后端终端会输出详细日志:
```
INFO:     127.0.0.1:xxxxx - "POST /auth/login HTTP/1.1" 200 OK
```
