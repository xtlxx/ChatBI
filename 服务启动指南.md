# ChatBI 服务启动指南

**最后更新**: 2026-01-21

---

## 🚀 快速启动

### 方式一: 使用一键启动脚本 ⭐ 推荐

**双击运行**: `启动服务.bat`

脚本会自动:
1. 激活虚拟环境
2. 在新窗口启动后端服务 (端口 8000)
3. 在新窗口启动前端服务 (端口 3000)

---

### 方式二: 手动启动

#### 启动后端:

```bash
# 1. 进入后端目录
cd backend

# 2. 激活虚拟环境
..\.venv\Scripts\activate

# 3. 启动服务
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

#### 启动前端:

```bash
# 1. 进入前端目录
cd frontend

# 2. 启动服务
npm run dev
```

---

## 🛑 停止服务

### 方式一: 使用停止脚本

**双击运行**: `停止服务.bat`

自动停止所有占用 8000 和 3000 端口的进程。

### 方式二: 手动停止

在运行服务的终端窗口按 `Ctrl + C`

## ❌ 常见问题

### 1. 端口被占用

**错误**: `Address already in use`

**解决**:
```bash
# 运行停止服务脚本
停止服务.bat

# 或手动查找并停止进程
netstat -ano | findstr :8000
taskkill /F /PID <进程ID>
```

### 2. 缺少依赖

**错误**: `No module named 'xxx'`

**解决**:
```bash
# 激活虚拟环境
.venv\Scripts\activate

# 安装依赖
cd backend
pip install -r requirements.txt

cd ../frontend
npm install
```

### 3. 数据库连接失败

**错误**: `Can't connect to MySQL server`

**解决**:
1. 检查 MySQL 服务是否启动
2. 检查 `backend/.env` 中的数据库配置
3. 确认数据库用户名和密码正确

### 4. greenlet 错误

**错误**: `No module named 'greenlet'`

**解决**:
```bash
.venv\Scripts\pip install greenlet
```

## 🔧 开发模式 vs 生产模式

### 开发模式 (当前)

**特点**:
- 热重载 (代码修改自动重启)
- 详细的错误信息
- 开发工具可用

**启动命令**:
```bash
# 后端
uvicorn app:app --reload --host 0.0.0.0 --port 8000

# 前端
npm run dev
```

### 生产模式

**后端**:
```bash
# 使用多进程
uvicorn app:app --host 0.0.0.0 --port 8000 --workers 4

# 或使用 Gunicorn
gunicorn app:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

**前端**:
```bash
# 构建生产版本
npm run build

# 启动生产服务器
npm start
```

## 📊 性能优化建议

### 后端优化:

1. **启用缓存**:
   - 在 `.env` 中配置 Redis
   - 设置 `CACHE_TTL=3600`

2. **数据库连接池**:
   - `DB_POOL_SIZE=10`
   - `DB_MAX_OVERFLOW=20`

3. **启用监控**:
   - `ENABLE_METRICS=true`
   - 访问 http://localhost:8000/metrics

### 前端优化:

1. **生产构建**:
   ```bash
   npm run build
   ```

2. **启用压缩**:
   - Next.js 自动启用 gzip

3. **CDN 加速**:
   - 部署静态资源到 CDN

## 🔐 安全建议

### 开发环境:

- ✅ 使用 `.env` 文件管理敏感信息
- ✅ 不要提交 `.env` 到 Git
- ✅ 定期更新依赖

### 生产环境:

- ⚠️ 更换所有默认密钥
- ⚠️ 启用 HTTPS
- ⚠️ 配置防火墙
- ⚠️ 启用 API 认证
- ⚠️ 限制 CORS 源

## 🆘 获取帮助

如果遇到问题:

1. **查看日志** - 检查终端输出的错误信息
2. **检查配置** - 确认 `.env` 文件配置正确
3. **重启服务** - 停止后重新启动

5. **重装依赖** - 删除 `node_modules` 和 `venv`,重新安装
